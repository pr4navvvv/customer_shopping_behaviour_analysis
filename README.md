# Customer Shopping Behavior Analysis

## Project Overview
This project analyzes customer shopping behavior using transactional data from **3,900 purchases** across multiple product categories. The objective is to identify patterns in **customer spending habits, product preferences, and subscription behavior** to help businesses make data-driven decisions.

The project combines **Python for data cleaning**, **SQL for analysis**, and **Power BI for visualization**.

According to the project document, the dataset contains **18 features** including demographics, purchase details, and behavioral metrics. :contentReference[oaicite:0]{index=0}


---

## Objectives
The main goals of this project are:

- Understand customer purchasing behavior
- Identify high performing products
- Analyze impact of discounts on sales
- Compare subscribers vs non-subscribers
- Segment customers into meaningful groups
- Build an interactive dashboard for insights


---

## Dataset Description

The dataset includes information related to customers and their shopping activity.

### Dataset Details
- Total Rows: **3,900**
- Total Columns: **18**
- Missing Values: **37 values in Review Rating column**

### Key Features
1. Customer Demographics
   - Age
   - Gender
   - Location
   - Subscription Status

2. Purchase Information
   - Item Purchased
   - Category
   - Purchase Amount
   - Season
   - Size
   - Color

3. Shopping Behaviour
   - Discount Applied
   - Promo Code Used
   - Frequency of Purchases
   - Previous Purchases
   - Review Rating
   - Shipping Type

Missing values in the review rating column were handled using median imputation for better accuracy. :contentReference[oaicite:1]{index=1}

## Project Workflow

### 1. Data Cleaning using Python

Data preprocessing was performed to improve data quality and consistency.

Key steps:
- Loaded dataset using pandas
- Used `.info()` and `.describe()` for initial exploration
- Handled missing values
- Renamed columns for consistency
- Created new features:
  - age_group
  - purchase_frequency_days
- Removed redundant columns like promo_code_used
- Loaded cleaned dataset into PostgreSQL

These steps ensured the dataset was ready for SQL analysis. :contentReference[oaicite:2]{index=2}


---

### 2. SQL Analysis (Business Questions)

SQL queries were used to extract business insights.

Key analysis performed:

1. Revenue by Gender
   - Compared total revenue generated by male vs female customers

2. High Spending Discount Users
   - Identified customers who used discounts but still spent above average

3. Top 5 Products by Rating
   - Found highest rated products

4. Shipping Type Comparison
   - Compared purchase amounts for standard vs express shipping

5. Subscriber vs Non Subscriber Analysis
   - Compared revenue contribution and average spending

6. Discount Dependent Products
   - Identified products mostly purchased using discounts

7. Customer Segmentation
   - Segmented customers into:
     - New
     - Returning
     - Loyal

8. Top 3 Products per Category
   - Identified most frequently purchased products per category

9. Repeat Buyers vs Subscription
   - Checked whether frequent buyers subscribe more

10. Revenue by Age Group
   - Identified which age group contributes most revenue

These SQL queries helped answer real business questions. :contentReference[oaicite:3]{index=3}

## Key Insights

- Loyal customers contribute a major portion of revenue
- Certain products rely heavily on discounts
- Clothing category generated highest sales
- Young adult customers contribute significant revenue
- Subscription users show consistent purchase behaviour
- Express shipping customers tend to spend slightly more

Insights were derived from SQL queries and Power BI dashboard visualizations. :contentReference[oaicite:4]{index=4}
